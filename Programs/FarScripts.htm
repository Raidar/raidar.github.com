LuaFAR

Библиотека LuaFAR расширяет возможности FAR Manager, позволяя писать плагины, скрипты и утилиты на языке Lua.

Скрипты обычно реализуют простые действия, утилиты — более сложные, комплексные. Скрипты и утилиты обычно объединяются в наборы и пакеты в соответствии с предпочтениями их разработчиков.

Начиная с FAR v.3.0 build 2851 библиотека вошла в состав FAR Manager, а язык Lua заменил использовавшийся до этого встроенный язык макросов.

 

Библиотека LuaFAR, плагины LuaFAR for Editor, LuaFAR Search, LuaFAR History, а также ряд других плагинов и макросов разработаны Сэмуэлом Зейгерманом (Shmuel Zeigerman).

Плагины

LuaFAR for Editor (LuaFAR для Редактора) — плагин для выполнения различных действий в редакторе FAR Manager (и не только в редакторе), влючает в себя набор полезных скриптов.

LuaFAR History (LuaFAR История) — плагин для отображения историй команд, папок и редактирования/просмотра файлов.

LuaFAR Search (LuaFAR Поиск и Замена) — плагин для поиска и замены текста в редакторе и на панелях FAR Manager с использованием различных регулярных выражений (Lua regex, FAR regex, PCRE, Oniguruma).

Пакеты

Пакет LF context

LuaFAR context — пакет для определения типов файлов и работы с файлами конфигураций параметров, связанных с типами файлов.

Пакет позволяет задать единую конфигурацию типов файлов в разных скриптах, плагинах и макросах LuaFAR, а также использовать в них конфигурации, зависящие от типа текущего файла.

Пакет разработан на основе конфигураций, используемых в fl_scripts, доработан и расширен совместными усилиями с maxfl . LF context требуется для работы пакета Rh_Scripts, пакета LuaFAR area config, и набора fl_scripts.

 

Для работы сборки пакета требуется:

  • FAR Manager v.3.0 build 3861.

Ссылка: сборка пакета v.3.1.0.3 [Build от 2014-05-11] (0,077 MiB).

Внимание: Перед установкой обязательно прочтите context\doc\install.rus. Для "LuaFAR для Редактора" пакет достаточно установить в каталог плагина. Пакет можно установить в каталог, доступный для всех скриптов LuaFAR. Порядок задания такого каталога описан в файле справки context\faq\faq_rus.hlf, тема «Где размещать общие скрипты и библиотеки?».

Пакет Rh_Scripts

Rh_Scripts — пакет скриптов и утилит для "LuaFAR для Редактора".

Состав:

  • Утилита «Пользовательское меню Lua» (LUM).
  • Утилита «Прямоугольное меню» (RectMenu).
  • Общие скрипты:
    • Скрипт «Настраиваемое меню» (CustomMenu).
    • Скрипт «Фильтрационное меню» (FilterMenu).
    • Скрипт «Вызыватель меню выбранного вида» (MenuCaller).
    • Скрипт «Формирователь текста для пунктов меню» (MenuTexter).
    • Скрипт «Календарь» (Calendar, см. календари).
    • Скрипт «Таблица символов» (CharsMap, см. таблицу символов).
    • Скрипт «Меню из пунктов вывода символов» (CharsMenu, см. наборы символов).
    • Скрипт «Конфигуратор пакета Rh_Scripts» (rhsConfig).
  • Скрипты для редактора:
    • Скрипт «Текстовые шаблоны» (TextTemplate).
    • Скрипт «Завершение слов» (WordComplete).
    • Скрипт для использования действий (AutoActions и HandActions).
    • Скрипт «Усекатель пустоты» (VoidTruncate).
    • Скрипт для «Усекателя пустоты» в авторежиме (TruncAction).
  • Служебные скрипты:
    • Скрипт «Обработка клавиш» (Keys).
    • Скрипт «Выполнение действий» (Actions).
    • Скрипт «Lua‑ и LuaFAR‑функции» (Utils).
    • Скрипт «Управление диалогами» (Dialog).
    • Скрипт «Обработка текста в редакторе» (Editor).
    • Скрипт «Работа с макросами‑шаблонами» (Macro).
    • Скрипт «Работа с меню и пунктами меню» (Menu).
    • Скрипт «Обработка даты и времени» (DateTime).
    • Скрипт «Работа с пользовательскими типами данных» (Types).
    • Скрипт «Обработка символов» (Character).
    • Скрипт «Разбор данных по Unicode‑символам» (CharsData).
    • Скрипт «Список Unicode‑символов» (CharsList).
    • Скрипт «Некоторые Unicode‑символы и наборы символов» (CharsSets).
  • Остальные файлы:
    • Пользовательские настройки пакета сохраняются в каталог
      %FarProfile%\data\scripts\Rh_Scripts\.
    • Дополнительные файлы для пакета необходимо располагать в каталоге
      %FarProfile%\data\scripts\Rh_Scripts\data\.
    • Дополнительные файлы для некоторых скриптов пакета:
      • NamesList.txt — названия Unicode‑символов.
      • Blocks.txt — названия блоков (тематических групп) символов.

Утилита «Пользовательское меню Lua» позволяет выводить меню в зависимости от типа файла.

Возможности:

  • Привязка файла к типу с помощью пакета LuaFAR context.
  • Привязка типа к выводимому меню в lua‑файле.
  • Поддерживаемые меню: прямоугольное, фильтрационное и поисковое.
  • Виды пунктов: подменю, макросы, шаблоны, скрипты и команды.
  • Назначение нескольких комбинаций клавиш на пункты меню.
  • Поддержка настроек собственных меню LUM и локализации текста.

Примеры меню LUM:

  • LuaEUM — для редактора.
  • LuaPUM — для панелей.
  • LumFLS — для работы с fl_scripts.

Примеры общих подменю:

  • Дополнения (Addon scripts):
    • Таблица символов (выбор блока).
    • Календари (Calendars):
      • Григорианский календарь.
      • Календарь тысячелетий.
      • 16‑ричный календарь.
      • Проект‑минимум календаря
        (вненедельные дни: 29 февраля, 31 декабря).
      • Пернский календарь.
  • Разные скрипты (Other scripts):
    Общая информация, Применение LF context, Пример Hello, world!.
  • Примеры команд (Command samples):
    Показ File_id.diz различными способами.

Примеры подменю для редактора (LuaEUM):

  • Вставка шаблона (Template Insert).
    Готовые подменю для Lua, C/C++, Pascal, Html.
  • Наборы символов (Character kits):
    • Буквы (греческие, кириллица).
    • Математика (символы и знаки, различные индексы).
    • Пунктуация (символы‑знаки, пробелы и черты).
  • Кавычки/скобки (Quotes/brackets):
    • Закавычивание (Quote text).
    • Раскавычивание (Dequote text).
  • Lua‑скрипты (Lua scripts):
    • Чистка текста (Clear text).
    • Усечение пустоты (Truncate void).
    • Изменение символов (Change characters).
  • Субтитры для файлов ass/ssa, srt (текущая фраза).

Примеры подменю для панелей (LuaPUM).

  • Меню Git (Git menu, Extended log, Reset/revert, Tag).
  • Меню команд TortoiseSVN. (TortoiseSVN menu).

Подменю LumSVN «TortoiseSVN» позволяет выполнять почти все команды TortoiseSVN.

Особенности:

  • Аналог контекстного меню TortoiseSVN Проводника.
  • Работает только с текущим элементом активной неплагиновой панели.
  • Различаются: файл/папка, рабочая копия или нет, версионированный или нет.
  • Для работы используются TortoiseProc.exe и SubWCRev.exe из TortoiseSVN.
  • Настройки хранятся в файле
    %FarProfile%\data\scripts\Rh_Scripts\LumSVN\config\TortoiseSVN.cfg.

Иллюстрация подменю «TortoiseSVN»:

  • версионированные папка и файл,
  • папка и файл нерабочей копии,
  • неверсионированные папка и файл.

Утилита «Прямоугольное меню» позволяет выводить меню в несколько строк и столбцов.

Возможности:

  • Меню в виде таблицы из нескольких строк и столбцов.
  • Параметры настройки для меню в целом и для отдельных пунктов.
  • Маркирование цветом заданных символов текста пунктов меню.
  • Поддерживаемые меню: фильтрационное и поисковое (far2.searchmenu).
  • Внешний вид: список, главное меню, всплывающее меню и т.д.
  • Пример списка: Список завершения слова.
  • Примеры меню: Календари, Таблица символов, Наборы символов, Вставка шаблона, Git, TortoiseSVN.

 

Для работы сборки пакета требуется:

  • FAR Manager v.3.0 build 3861.
  • LuaFAR for Editor v.3.5.1 .
  • LF context v.3.1.0.3 .
  • для flsLUM — fl_scripts v.3.1.0.0+ .

Макрос LuaSpell

LuaSpell — макрос для проверки орфографии в редакторе FAR Manager.

Макрос использует библиотеку hunspell из проекта luapower.

 

Для работы макроса требуется:

  • FAR Manager v.3.0 build 3861.

Ссылка: сборка v.3.1.5.0 [Build от 2014-07-11] (0,011 MiB),
библиотека hunspell v.1.3.2 (0,233 MiB),
русский (с "ё") и английский словари в кодировке UTF‑8 (0,606 MiB),
старый пример конфигурации для tex‑тегов с заготовкой словаря (0,001 MiB).

Внимание:

  • Перед установкой макроса прочтите LuaSpell\doc\Install.rus.
  • Все файлы словарей должны быть в кодировке UTF‑8 без BOM, а в первой строке aff‑файлов нужно прописать SET UTF-8.

Пакет LFA config

LuaFAR area config — пакет для плагина "LuaFAR для Редактора", предназначенный для автонастройки редактора FAR в зависимости от типа файла.

Возможности:

  • Задание настроек редактора в зависимости от типа файла.
  • Показ номеров строк (слева от поля редактирования).
  • Подсветка позиции конца строки и конца файла.
  • Подсветка правой границы и последней строки файла.

Иллюстрация работы пакета: пример 1, пример 2.
